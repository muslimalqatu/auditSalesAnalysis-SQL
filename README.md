# Audit-Oriented Sales Data Analysis

## Overview
This project simulates a financial transactions environment and applies SQL-based audit analytics techniques to identify anomalies, risks, and data inconsistencies.

## Tools
- SQL (PostgreSQL)
- Python (for data generation)

## Project Structure
- sql/: database schema, data import, and audit queries
- data/: sample datasets
- scripts/: data generation scripts
- analysis/: audit findings and documentation

## Key Features
- Designed relational database schema
- Performed audit analytics using SQL
- Detected anomalies such as:
  - approval limit violations
  - duplicate payments
  - unusual transaction timing
  - high-risk vendor activity
- Documented findings in a structured format

## Key Insights
- Control violations can occur when approval limits are exceeded
- Duplicate payments can result from poor data validation
- Transactions outside business hours may indicate suspicious activity

## Conclusion
This project demonstrates the use of SQL in audit analytics and highlights the importance of data validation in financial systems.
